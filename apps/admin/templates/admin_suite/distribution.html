{# ═══ How Video Generation Works ═══ #}

1. AI Script: An AI generates a video script from the blog post — hook, scenes, CTA, and hashtags — tailored to the target platform.

2. Animated Presentation: A custom HTML presentation renders the script with MrBeast-style effects: bold text, gradient backgrounds, auto-paced scene transitions, progress bar.

3. Browser Recording: Playwright opens the presentation in a headless browser and records the viewport as video at native resolution.

4. Post-Processing: FFmpeg converts to MP4 H.264, optimizes for streaming, and extracts a thumbnail frame.

5. Platform Specs: YouTube (1920×1080 cinematic) · YouTube Shorts (1080×1920 fast) · TikTok (1080×1920 trending) · Instagram Reels (1080×1920 visual) · Facebook (1280×720 story) · LinkedIn (1920×1080 professional) · Twitter (1280×720 punchy).

6. Auto-Publish: Check "Auto-publish when ready" to queue videos for immediate upload to configured platform accounts.

Prerequisites: Playwright (headless Chromium) and FFmpeg must be installed. Run pip install playwright && playwright install chromium and install FFmpeg from ffmpeg.org.

{# How Auto-Publishing Works #}

1. Create Accounts: Go to Accounts tab → Add Account → Select a platform → Enter your API keys/tokens/credentials → Save.

2. Enable Distribution: Go to Settings → Distribution Settings → Enable "Distribution Enabled" and "Auto-Fanout on Publish".

3. Auto-Fanout: When a blog post is published, the system automatically creates a SharePlan with distribution jobs for each active platform account.

4. Job Execution: Celery workers pick up pending jobs, format content using each platform's template, and post via the platform's API.

5. Credential Types by Platform:

  • Telegram: Bot Token (from @BotFather) + Chat/Channel ID
  • Discord: Webhook URL (Channel Settings → Integrations)
  • Medium: Integration Token (Settings → Security)
  • WhatsApp: Business API Access Token
  • Twitter, Facebook, LinkedIn, Instagram, Reddit, Pinterest: OAuth2 — API Key + API Secret (create dev app on each platform)

6. Monitoring: Check Jobs tab for pending/failed jobs. Failed jobs can be retried. Check Logs tab for detailed delivery logs.
